Troubleshooting Common Issues

Issue: Migration fails with "table already exists"

Solution
This is expected if you've already run the migration. Run verification script instead:
pnpm tsx scripts/verify-implementation.ts

Issue: Seed script says permissions already exist

Solution
Permissions are already seeded. This is normal. Run verification to confirm:
pnpm tsx scripts/verify-implementation.ts

Issue: API endpoints returning 404

Solution
1. Verify routers are registered in server/routers.ts
2. Restart the development server
3. Check browser console for errors

Issue: Verification script returns errors

Solution
1. Check .env database credentials
2. Verify MySQL connected: mysql -u root -p
3. Verify migrations applied: pnpm run db:push
4. Verify seed script ran: pnpm tsx scripts/seed-permissions.ts
